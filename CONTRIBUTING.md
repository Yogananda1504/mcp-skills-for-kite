# Contributing

Thanks for helping improve this skills library.

## What to contribute

- New skills that fit the repository scope.
- Improvements to existing skill instructions.
- Documentation fixes and workflow updates.
- Bug fixes in the publishing setup or repository metadata.

## Before you open a pull request

1. Keep skills in `skills/<skill-name>/SKILL.md`.
2. Keep frontmatter valid and the `name` aligned with the folder name.
3. Update `README.md` if the repository structure or publishing flow changes.
4. Run or review the publish workflow impact before merging.

## Pull request expectations

- Keep changes focused and easy to review.
- Explain the user-facing effect of the change.
- Call out any publishing or compatibility implications.

## Style guidance

- Use clear, direct language.
- Keep skill instructions specific and actionable.
- Prefer small, scoped updates over broad rewrites.
