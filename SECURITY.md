# Security Policy

## Supported content

This repository contains skill definitions and GitHub Actions workflow files. We treat
publishing workflow integrity and repository metadata as security-sensitive.

## Reporting a vulnerability

If you discover a security issue, do not open a public issue. Instead, report it through
GitHub's private vulnerability reporting channel for the repository or contact the
maintainers directly.

## What to include

- A short description of the issue.
- The file or workflow path involved.
- The impact and any reproduction steps.
- Suggested mitigation if you have one.

## Scope

Please report issues involving:

- Workflow credential handling.
- Unexpected publishing behavior.
- Malicious or unsafe skill metadata.
- Any repository content that could cause unsafe execution or distribution.
